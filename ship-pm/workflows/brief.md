# Workflow: Feature Context

## 1. Validate Target Feature

Extract the Feature number from `$ARGUMENTS`. Let's call it `$FEATURE`.

Read `.pm/ROADMAP.md` to confirm the Feature exists.
Extract the Feature name and goal.

Example:
If `$FEATURE` is `1`, find `### Feature 1: [Name]` and look for `**Goal**:`.
Create a slugified version of the Feature name (e.g., `user-authentication`) and pad the Feature number (e.g., `01`, `02.1`).

Let `$FEATURE_DIR` = `.pm/briefs/${PADDED_FEATURE}-${SLUG}`.
Create this directory if it doesn't exist:
```bash
mkdir -p "$FEATURE_DIR"
```

## 2. Load Global Context

Read the following to understand the rules of the system:
- `.pm/PROJECT.md` — overall technical stack and constraints
- `.pm/SCOPE.md` — all features in v1 scope
- `.po/features/` — any detailed feature specs generated by `/pm:refine` that match this Feature's goals.

## 3. Generate -BRIEF.md using PM Brief Writer

Use `@~/.claude/ship-pm/agents/pm-brief-writer.md` to synthesize the raw Feature goal and the global context into a highly opinionated technical brief.

Write the output directly to: `${FEATURE_DIR}/${PADDED_FEATURE}-BRIEF.md`

The format of this file MUST follow this robust `BRIEF.md` structure so that it seamlessly constrains the downstream execution agent:

```markdown
# Feature [X]: [Name] - Brief

**Gathered:** [date]
**Status:** Ready for planning
**Source:** PM Assistant `/pm:brief`

<domain>
## Feature Boundary
[Exactly what this Feature delivers, synthesized from ROADMAP and SCP]
</domain>

<decisions>
## Implementation Decisions

### [Category 1: e.g. Data Model]
- [Locked decision: "Use users table with auth_provider field"]

### [Category 2: e.g. User Flow]
- [Locked decision: "Redirect to /dashboard after login"]

### Claude's Discretion
[Areas where the developer should use their best judgment. Mention anything explicitly NOT decided here]

</decisions>

<canonical_refs>
## Canonical References
[Paths to any feature-specs or existing architectural decisions that downstream agents MUST read before planning]
</canonical_refs>

<specifics>
## Specific Ideas
[Any code snippets, library recommendations, or API structures desired]
</specifics>

<deferred>
## Deferred Ideas
[Things that are explicitly out of scope for this Feature]
</deferred>

---
*Generated by PM Assistant*
```

## 4. Output

Display a success banner and handoff instructions.

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
 PM ► -BRIEF GENERATED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

 Wrote: ${FEATURE_DIR}/${PADDED_FEATURE}-BRIEF.md

 Summary:
 - Boundary: [1 sentence overview]
 - Key Decisions: [1-2 main decisions locked in]
 - Deferred: [What was explicitly left out]

 Next Step: 
 Run your execution agent to execute the Feature:
 [agent command]
```
